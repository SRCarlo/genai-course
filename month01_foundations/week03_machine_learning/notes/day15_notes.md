# Introduction

Machine Learning (ML) is a branch of Artificial Intelligence that allows computers to learn patterns from data and make predictions or decisions without being explicitly programmed.

Examples:

* Predict student scores
* Predict house prices
* Recommend movies
* Detect spam emails
* Predict customer purchases

Machine Learning projects follow a structured workflow to ensure accurate and reliable results.

# 1. Problem Definition

## What is Problem Definition?

Problem Definition is the process of clearly identifying the problem that needs to be solved.

Before writing code, ask:

* What problem are we solving?
* What data do we need?
* What should the model predict?

### Example

Problem:

Predict student scores based on study hours.

Input:

Study Hours

Output:

Expected Score

### Importance

A well-defined problem helps:

* Collect the right data
* Select the right model
* Measure success properly

---

# 2. Data Collection

## What is Data Collection?

Data Collection is the process of gathering information that will be used to train the machine learning model.

Machine Learning learns from data.

No Data = No AI

### Sources of Data

* CSV Files
* Databases
* APIs
* Sensors
* User Activity
* Internet Data

### Example Dataset

| Hours | Score |
| ----- | ----- |
| 1     | 15    |
| 2     | 25    |
| 3     | 35    |
| 4     | 45    |

### Importance

High-quality data usually produces better predictions.

---

# 3. Data Cleaning

## What is Data Cleaning?

Data Cleaning is the process of removing or correcting bad data.

### Common Problems

* Missing Values
* Duplicate Records
* Incorrect Values
* Invalid Entries
* Outliers

### Example

Before:

[10, 20, None, 40, 50]

After:

[10, 20, 40, 50]

### Importance

Garbage In = Garbage Out

Poor quality data leads to poor model performance.

---

# 4. Feature Engineering

## What is a Feature?

A Feature is an input variable used by a machine learning model.

### Examples

Student Prediction:

Features:

* Study Hours
* Attendance
* Assignment Marks

Target:

* Exam Score

### Good Features

* Hours Studied
* Attendance
* Salary
* Experience

### Poor Features

* Random IDs
* Student Name

### Importance

Better features often improve model accuracy significantly.

---

# 5. Train-Test Split

## What is Train-Test Split?

Train-Test Split divides data into:

* Training Data
* Testing Data

### Common Split

* 80% Training
* 20% Testing

### Example

100 Records

80 → Training

20 → Testing

### Why Split Data?

Testing data helps evaluate model performance on unseen data.

Without testing:

The model may simply memorize training data.

---

# 6. Model Training

## What is Model Training?

Training is the process of learning patterns from historical data.

### Example

Hours → Score

2 → 20

5 → 50

8 → 80

Pattern:

Score = Hours × 10

### Goal

Learn relationships between inputs and outputs.

---

# 7. Model Evaluation

## What is Evaluation?

Evaluation measures how well the model performs.

### Example

Actual Score = 80

Predicted Score = 75

Error = 5

### Questions Answered by Evaluation

* Is the model accurate?
* Can it predict unseen data?
* Is improvement needed?

### Common Metrics

Regression:

* MAE
* MSE
* RMSE

Classification:

* Accuracy
* Precision
* Recall
* F1 Score

# Overfitting

## Definition

Overfitting occurs when a model memorizes training data instead of learning patterns.

### Characteristics

* High Training Accuracy
* Low Testing Accuracy

### Example

A student memorizes answers without understanding concepts.

### Result

Poor performance on new data.

---

# Underfitting

## Definition

Underfitting occurs when a model fails to learn important patterns.

### Characteristics

* Low Training Accuracy
* Low Testing Accuracy

### Example

A student studies very little before an exam.

### Result

Poor predictions everywhere.

A Machine Learning Workflow is a structured process used to build machine learning models from problem definition to deployment.

Steps:

1. Problem Definition
2. Data Collection
3. Data Cleaning
4. Feature Engineering
5. Train-Test Split
6. Model Training
7. Evaluation

Problem Definition helps identify:

* The objective of the project
* Required data
* Expected output

A clear problem statement guides the entire workflow.

* APIs
* Sensors

Data Cleaning removes missing, incorrect, and duplicate records.
